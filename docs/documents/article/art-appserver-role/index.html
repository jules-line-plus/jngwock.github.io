    AppServer Role

    1 minute read

    To setup a 1:1 call or a group call, AppServer should make a callee(or participants) perceive.

    PlanetKit call requires proper information to communicate, so AppServer should to deliver some fields depends on call type.

    • 1:1 Call
    • Group Call

    1:1 Call - cc_param

    AppServer MUST deliver cc_param to a callee that is copied from param in Notify (notify_cb).

    Data from notfy_cb Type Corresponding push payload key
    param String cc_param


    You can use mobile push system to accomplish this as almost mobile apps do.

    Here we assume you use APNs and FCM push system.

    Also you can define application’s push data using other fields from notify_cb to handle application requirements such as showing caller’s information or caller’s call type.

    Here show examples.
    “app_” prefix means defined by application which means they are not required by PlanetKit call setup.

    Group Call

    Unlike 1:1 call, there is no requirements for AppServer to setup a group call.
    Just be sure that all participants should share the same service id and conference room id to communicate in the same room.

    An application can share them via the application specific channel without involving handling of Group Call Event (on_gcall_evt_cb) depends on an application group call setup flow.

    If an application hope to use on_gcall_evt_cb, the service id and room id can be copied from following fields

    • service id: svc_id
    • room id: id
